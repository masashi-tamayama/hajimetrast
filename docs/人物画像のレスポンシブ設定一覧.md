# 人物画像（680918.png）レスポンシブ設定一覧（最新版）

## 実装方針

**目標**: 1920pxから縮小する際、ファーストビュー全体の枠が自然に小さくなり、中のパーツ（人物、テキスト、吹き出し、メダル等）も一緒にスムーズに縮小する。

**アプローチ**:
- 固定の`vh`値を排除し、`min-height`で柔軟に対応
- 人物画像は`max-height: Nvh`で高さ制限し、画面縮小に連動
- テキストやカードは`clamp()`で流動的にサイズ調整
- 吹き出し・メダルは`vw`や`clamp()`で相対サイズ

## 動作の特徴

### 自然な縮小の仕組み

1. **画面幅1920px → 1440px → 1280px → 1024px と縮小する際:**
   - `section-mv2`の`min-height`が`85vh → 80vh → 75vh → 70vh`と段階的に減少
   - 人物画像の`max-height`も同じく減少し、人物が垂直方向にも小さくなる
   - 人物ラッパーの`width`も`45% → 42% → 40% → 38%`と減少し、横幅も縮小

2. **テキストやカードは`clamp()`で自動調整:**
   - 画面幅が狭まるほどフォントサイズやカードサイズが滑らかに縮小
   - 最小値・最大値の範囲内で`vw`（ビューポート幅）に比例

3. **吹き出し・メダルも`vw`基準で連動:**
   - `clamp(200px, 20vw, 350px)`のように幅に応じてサイズ変化
   - 人物や他のパーツと同じペースで縮小

4. **結果:**
   - ファーストビュー全体が**自然にスムーズに縮小**
   - 各パーツの相対的なバランスを維持
   - 固定サイズによる破綻や、特定の幅でのレイアウト崩れを回避

## 従来の問題点と解決策

### 従来の問題

1. **固定`height: 85vh`**: フレームが縦方向に固定され、画面縮小時に人物が下に伸びる
2. **人物`height: 100%`**: ラッパーの高さ100%で固定され、アスペクト比が崩れる
3. **固定px値の吹き出し・メダル**: 画面縮小時にバランスが崩れる
4. **固定フォントサイズ**: 画面縮小時にテキストが大きすぎる/小さすぎる

### 解決策

1. **`min-height`に変更**: 内容に応じて伸縮可能
2. **人物は`height: auto` + `max-height: Nvh`**: アスペクト比維持しつつ、画面縮小に連動
3. **`clamp()`で相対サイズ**: 全要素が画面幅に応じて滑らかに変化
4. **段階的なメディアクエリ**: 各ブレークポイントで`vh`値を調整し、自然な縮小を実現

## 結論

この実装により、1920pxから縮小する際に**ファーストビュー全体、人物画像、中央コンテンツがすべて同時に自然にスムーズに縮小**し、余計な要素を増やすことなく、レスポンシブ対応が完成しました。
